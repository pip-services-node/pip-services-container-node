  • new ContainerReferences(tuples?: any[]): ContainerReferences

  • Inherited from ManagedReferences.constructor

    Overrides ReferencesDecorator.constructor

    • Defined in refer/ManagedReferences.ts:23

    Creates a new ManagedReferences object using the provided tuples array of references.

    This constructor passes the tuples array to References' constructor, the result of which is passed to BuildReferencesDecorator's constructor (the created "References" are passed as nextReferences and this object itself is passed as "parent references"), the result of which is passed to LinkReferencesDecorator's constructor, the result of which is passed to RunReferencesDecorator's constructor, the result of which is set as this object's base references.

putFromConfig

  • putFromConfig(config: ContainerConfig): void
    • Defined in refer/ContainerReferences.ts:56

    Adds component configurations to this ContainerReferences object.

    For each component configuration in the passed container configuration the following is done:

    • References are set:
      • if the ComponentConfig contains a "type" - an attempt will be made to create the component dynamically.
      • otherwise, if the ComponentConfig contains a "descriptor" - an attempt will be made to create the component using the stored locator-Descriptor.
      • if the component was successfully created - it is added to this object's list of references.
    • The component is setup:
      • if the component is configurable - it will be configured using the ComponentConfig's "config".
      • if the component can reference other components (factories) - it will be set to reference this ContainerReferences object.
    see

    ContainerConfig

    see

    ComponentConfig

    see

    Descriptor (in the PipServices "Commons" package)

    see

    IConfigurable (in the PipServices "Commons" package)

    see

    IReferenceable (in the PipServices "Commons" package)

    throws

    a CreateException if the component cannot be created using one of the ContainerConfig's ComponentConfig's (explicit) "type" or "descriptor" (locator).

    throws

    a ReferenceException if an object cannot be created by one of the ContainerConfig's ComponentConfig's descriptor (locator).

    throws

    a ReferenceException if any other exception is raised.

    Parameters

    • config: ContainerConfig

      the ContainerConfig, whose ComponentConfigs are to be added.

    Returns void
